# 🔍 Google Link (WME)

Tampermonkey скрипт для [Waze Map Editor](https://www.waze.com/editor), який автоматично шукає Google POI за адресою venue та прив'язує Google Place ID.

🔗 [GitHub](https://github.com/EdjOne/google-link) | 📦 [Встановити скрипт](https://raw.githubusercontent.com/EdjOne/google-link/master/src/wme-google-link.user.js)

---

## Що робить

Скрипт з'єднує WME venues з Google Maps POI. Коли ти клікаєш на POI в WME, скрипт:

1. **Отримує адресу** venue через WME SDK (вулиця, номер, місто, країна)
2. **Шукає в Google Places** через AutocompleteService + PlacesService
3. **Показує результати** у плаваючій панелі з адресою, телефоном, рейтингом та відстанню
4. **Прив'язує** Google Place ID до venue одним кліком

### Альтернативні назви вулиць

Скрипт підтримує **альтернативні (старі) назви вулиць** з WME. Якщо venue знаходиться на вулиці з історичною назвою, скрипт спробує знайти Google Place і за оригінальною, і за новою назвою.

## Встановлення

### Крок 1: Tampermonkey

Встанови розширення [Tampermonkey](https://www.tampermonkey.net/) для свого браузера (Chrome, Firefox, Edge, Opera).

### Крок 2: Скрипт

1. Перейди на сторінку [скрипта на GitHub](https://github.com/EdjOne/google-link/blob/master/src/wme-google-link.user.js)
2. Натисни кнопку **"Raw"** (сіра кнопка праворуч)
3. Tampermonkey запропонує встановити — натисни **"Встановити"**

### Крок 3: WME

1. Відкрий [Waze Map Editor](https://www.waze.com/editor)
2. У лівій панелі з'явиться вкладка **🔍 GL** — це панель скрипта

## Як користуватися

### Пошук та перегляд

1. **Клікни на POI** (venue) на карті WME
2. З'явиться плаваюча панель **"🔍 Google Link"** справа
3. Скрипт автоматично шукає в Google за адресою venue
4. **Результати** відображаються з кольоровою індикацією відстані:
   - 🟢 **< 50 м** — точно те саме місце
   - 🟡 **50–300 м** — поруч, можливо те саме
   - 🔴 **> 300 м** — далеко, ймовірно інше місце

### Прив'язка

1. **Клікни на результат** Google в панелі
2. Скрипт автоматично:
   - Відкриє секцію "Зовнішні сервіси" (External Providers) в редакторі
   - Натисне кнопку **"Прив'язати до Google"**
   - Заповнить поле пошуку адресою
   - Обере перший варіант
3. **Перевір** правильність прив'язки
4. **Збережи** зміни (Ctrl+S)

### Перетягування панелі

Панель можна перетягувати за заголовок — перемісти її в зручне місце на екрані.

## Налаштування (вкладка 🔍 GL)

| Налаштування | Опис |
|---|---|
| **📍 Показувати відстань** | Додає кольорову індикацію відстані до кожного результату (🟢🟡🔴) |
| **🔗 Тільки без посилань** | Підсвічує cyan на карті тільки POI, які ще не прив'язані до Google |
| **Радіус: ___ м** | Максимальна відстань пошуку (за замовчуванням 5000 м) |

### Підсвітка нелinkованих POI

Якщо ввімкнено **"🔗 Тільки без посилань"**:

- 🔵 **Іконки** на карті отримують cyan-обводку (stroke)
- 🔵 **Текстові мітки** стають cyan + жирний + свічення
- Подсвітка **оновлюється автоматично** при зумі, переміщенні карті, збереженні POI

## Фільтри (що пропускається)

Скрипт автоматично **ігнорує** певні категорії venues:

### 🌿 Природа (NATURAL_FEATURES)

- Острови (ISLAND)
- Озера/ставки/басейни (SEA_LAKE_POOL)
- Річки/струмки (RIVER_STREAM)
- Ліси/гаї (FOREST_GROVE)
- Ферми (FARM)
- Канали (CANAL)
- Болота/твердині (SWAMP_MARSH)
- Плотини (DAM)

### 🅿️ Парковки (PARKING_LOT)

Всі парковки без підкатегорій.

**Ці POI:**
- ❌ Не підсвічуються при "Тільки без посилань"
- ❌ Не відкривають панель Google Link при кліку
- ✅ Звичайні venues (кафе, магазини, офіси тощо) працюють нормально

## Технічні деталі

### Джерела даних

- **WME SDK** — отримання адреси venue, координат, назви вулиці
- **Google Places API** — AutocompleteService + PlacesService (без ключа)
- **WME legacy model** — альтернативний шлях для отримання атрибутів

### Як працює прив'язка

1. Знаходить DOM-елемент кнопки "Прив'язати до Google" в shadow DOM WME
2. Імітує клік для відкриття панелі зовнішніх сервісів
3. Знаходить input пошуку Google Places (також в shadow DOM)
4. Заповнює input посимвольно для активації автокомпліту
5. Чекає на появу dropdown (pac-container) та обирає перший варіант

### Альтернативні назви вулиць

- Отримує `streetIDs` з сегмента venue
- Знаходить альтернативні вулиці через `W.model.streets`
- Додає їх до пошукового запиту Google

---

## Обмеження

- Працює тільки з **POI** (venue), не з адресними точками/шляхами
- POI **має мати адресу** (вулиця + номер будинку)
- Google Places API без ключа — обмеження ~100 запитів/день
- Після прив'язки — **збережи** зміни вручну (Ctrl+S)
- Не працює з residential venues та placeholder'ами
